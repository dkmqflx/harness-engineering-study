오늘의 논제

각 글이 밝힌 사실에서 한 걸음 더 — 아직 안 풀린 질문으로 토론을 엽니다. 배경 사실이 궁금하면 각 논제의 근거 보기를 펼치세요.

01

CLAUDE.md 규칙, 어디에 어떻게 쓸까

김도현

부정형('하지 마라')이 오히려 금지한 행동을 부른다면, 어떤 규칙은 긍정형으로 바꾸고 어떤 규칙은 그대로 둬야 할까?

아티클이 밝힌 것 Do NOT add comments 지시가 오히려 주석을 유발했다(Pink Elephant 효과). 다만 보안 규칙에서는 부정형이 더 신중한 응답을 끌어낸 단일 관찰도 있었다.

규칙을 분리했을 때 손해였던 건 분리 방식 자체의 한계일까, 아니면 짧은 비대화형 태스크라는 실험 조건 탓일까?

아티클이 밝힌 것 @import 없이 docs/만 참조하면 파일을 읽지 않아 규칙 준수율이 떨어졌고, 토큰 절감도 없었다 (thin 조건 Task 1이 fat의 4.3배, 161,668 토큰). 실험은 claude --print 비대화형·단기 태스크였다.

규칙이 안 지켜질 때 진짜 해법은 규칙 수를 줄이는 것일까, 아니면 늘 보이게(항상 로드되는 컨텍스트에) 두는 것일까?

아티클이 밝힌 것 분리는 준수율·토큰 모두 이득이 없었다. Anthropic은 규칙 수 줄이기를 권고하지만, 반대 선택지로 큰 CLAUDE.md나 @import로 항상 로드하는 방법이 있다.
02

점수를 목표로 주면 UI를 망칠까

김찬영

점수는 올랐는데 UI가 망가진 걸 '점수만 좇은 탓'으로 단정할 수 있을까? 그리고 UI 훼손은 점수처럼 어떻게 재야 할까?

아티클이 밝힌 것 Lighthouse 95점 이상만 제시하자 99.7점까지 올렸지만 사람이 보기 편한 UI를 망쳤다(Goodhart's Law 양상). UI 훼손 판정은 작성자 1인의 시각 평가였다.

목표를 안 줬을 때 결과가 더 나았던 건 목표 부재 덕분일까, 아니면 점수가 낮아 덜 건드렸을 뿐일까?

아티클이 밝힌 것 점수 목표가 없던 시나리오 C는 평균 92점(95점 미달)에 멈췄지만 결과물은 오히려 더 나았다.

보존할 것을 충분히 나열하면 점수와 UI를 양립시킬 수 있을까, 아니면 나열하지 못한 부분은 계속 망가질까?

아티클이 밝힌 것 UI 보존 제약을 명시한 시나리오 D는 96.7점에 도달했다(점수와 UI를 함께 잡은 사례).
03

에이전트도 나쁜 코드 습관을 따라 할까

김시온

이걸 에이전트가 '나쁜 습관을 따라 한' 거라고 봐야 할까, 아니면 주변 코드의 타입·구조에 단순히 반응한 것뿐일까?

아티클이 밝힌 것 더러운 코드베이스 위에서 새 기능을 시키면 안티패턴 재현율이 깨끗한 코드베이스보다 높아졌다 (6.1%→22.7%).

콕 집어 지시했을 때 줄어든 건 지시의 효과일까, 아니면 측정 항목과 지시 항목이 겹쳐 생긴 착시(동어반복)일까?

아티클이 밝힌 것 콕 집은 사전 지시로 표면적 안티패턴은 줄었다. 그런데 팀 표준이 측정 대상 안티패턴 6종을 그대로 짚고 있었다(측정=지시 겹침).

자기점검으로도 끝까지 안 고쳐지는 패턴은 무엇이 다를까? 무엇이 '끝까지 남는지'를 가를까?

아티클이 밝힌 것 작성 후 자기점검은 재현율을 29%→15%로 낮췄지만, 가장 깊이 밴 패턴(디자인 토큰 등)은 100%→100%로 그대로였다.

교차 논제

세 실험을 가로지르는 질문

목표·금지어·주변 코드처럼 에이전트에 준 신호가 의도와 다른 행동을 부른다면, 이런 신호를 미리 알아챌 공통 기준을 만들 수 있을까?

세 글에서 단일 지표 목표 → UI 훼손 · 부정형 지시 → 금지 행동 유발 · 더러운 주변 코드 → 안티패턴 모방.

표본·측정·채점이 글마다 제각각인데, 이 결과들을 어디까지 믿고 실무에 적용할 수 있을까? 무엇이 더 있으면 믿을 수 있을까?

세 글에서 실행 횟수(태스크당 1회 / 15회 / 96회), 측정 자동화 여부, 채점자 독립성이 글마다 제각각이다.
