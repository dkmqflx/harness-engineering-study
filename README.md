# 하네스 엔지니어링 스터디

코딩 에이전트(하네스)를 더 잘 쓰는 법을 매주 글로 검증하고, 모여서 토론하는 스터디입니다.
컨텍스트 엔지니어링, 워크플로우, 서브에이전트·오케스트레이션, 평가·측정 등을 다룹니다.

## 진행 방식

1. **아티클 작성** — 관심 주제로 마크다운 아티클 1편을 씁니다. 주제 자유, 분량 무제한.
2. **사전 공개** — 작성한 아티클을 스터디 모임 전에 git에 올려 미리 공개합니다.
3. **오프라인 스터디** — 준비된 에이전트가 각자의 아티클을 기반으로 논제를 던지고(각자 궁금한 점을 제시해도 좋습니다), 그 논제를 바탕으로 함께 토론합니다.

## 일정

| 무엇 | 언제 | 어디서 |
| --- | --- | --- |
| 아티클 작성·업로드 | 매주 **일요일 14:00~24:00** | git |
| 오프라인 스터디 | 매주 **화요일 19:30** | 숙대입구역 근처 스터디룸 |

## 아티클 작성 규칙

- **검증 가능한 주장 하나를 반드시 포함**합니다 — `~하면 ~가 나아진다/줄어든다` 형태.
  - 예) "plan 모드로 먼저 탐색하면 환각 API 호출이 줄어든다", "claude.md를 얇은 인덱스로 두면 토큰이 줄어든다"
- **검증 기준**: 핵심 근거는 둘 중 하나여야 합니다.
  - (a) 본인이 **직접 돌려본 결과**, 또는
  - (b) 본인이 **직접 확인한 1차 출처**
- **AI 활용은 적극 권장**합니다. 초안을 잡거나 구조를 짜거나 표현을 다듬는 데 자유롭게 쓰세요.
  단, 글의 핵심 주장과 그걸 뒷받침하는 근거는 **본인이 직접 검증하고 설명할 수 있어야** 합니다.
  "AI가 이렇게 썼다"는 근거가 될 수 없습니다.

## 레포 구조 / 파일 컨벤션

아티클은 **그 주 일요일 날짜 폴더** 아래에 **작성자별 파일**로 둡니다.

```
articles/
  2026-06-07/                     # 그 주 일요일(작성일) 날짜
    sion-claude-md-index.md       # <작성자>-<슬러그>.md
    <name>-<slug>.md
  2026-06-14/
    ...
README.md
```

- 폴더명: 작성하는 주의 **일요일 날짜** `YYYY-MM-DD` (예: `2026-06-07`)
- 파일명: `<작성자>-<주제-슬러그>.md` (영문 소문자 + 하이픈 권장)
- 한 사람이 한 주에 한 편을 기본으로 합니다.

## 주제 예시

**컨텍스트 엔지니어링**
- claude.md를 얇은 인덱스로 두는 게 통짜로 다 넣는 것보다 나은가
- 휘발성 데이터는 MCP로, 안정적 지식은 파일로 나누면 환각이 줄어드나

**워크플로우**
- plan 모드(탐색→스펙→구현)가 바로 구현하는 것보다 환각 API를 줄이나
- 작업을 잘게 쪼개면 한 번에 큰 요청하는 것보다 재작업이 줄어드나

**서브에이전트·오케스트레이션**
- 검증 전용 서브에이전트를 따로 두면 버그를 더 잡나
- 멀티 에이전트가 단일 에이전트보다 실제 가치가 있나, 토큰만 더 쓰나

**평가·측정**
- AI 출력 신뢰성을 어떤 지표로 자동 채점할 수 있나 (테스트 통과율, 환각 여부, 제약 준수)
- 같은 과제를 다른 하네스로 돌리면 출력이 얼마나 갈리나
